# 📋 [결과 보고서] 소비재 전시회 참관기업 Lead Evaluation 및 실전형 CRM 대시보드 구축 프로젝트

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적
국내 소비재 전시회(뷰티, 푸드, 리빙 등)에는 매년 수많은 유통·제조 기업들이 참가하지만, 사후 리드(Lead) 관리가 체계적으로 이루어지지 않아 비즈니스 계약 성사율이 떨어지는 고질적인 문제가 있습니다. 
본 프로젝트는 **(1) 가상의 참관사 데이터를 정밀 생성하고, (2) 기업 가치를 다차원 가중치 모델로 자동 채점(Lead Scoring)하며, (3) 최종적으로 크롤링된 실시간 시장 데이터와 연계해 입체적인 비주얼을 제공하는 '실전형 CRM 대시보드'를 완벽 구현**하는 것을 목표로 수행되었습니다.

### 1.2 추진 성과 요약
* **안정적인 아키텍처 수립**: 샘플 생성기, 가중치 평가기, 데이터 크롤러, 시각화 프레젠테이션 레이어를 고도로 모듈화하여 구조화.
* **실시간 원천 데이터 획득**: 대한민국 대표 소비재 전시회인 '메가쇼(Mega Show)'의 실시간 브랜드 카탈로그 데이터를 합법적이고 안전하게 우회 수집(40개 실제 제조 브랜드 확보).
* **입체적인 CRM 대시보드 구현**: 가상 데이터와 실제 데이터를 스위칭하며 분석 가능한 Streamlit 배포 및 한국 산업 거점 12곳을 정합한 3D Mapbox 공간 시각화 완성.
* **자동 동기화 파이프라인**: GitHub 원격 리포지토리 및 Streamlit Cloud 연동을 통한 실시간 빌드/배포 자동화.

## 2. 세부 개발 방법론 (Methodology)

### 2.1 데이터 가상화 기법 (Data Generation Layer)
* **파일명**: `data/generate_sample_data.py`
* **방법론**: 현실적인 대한민국 소비재 유통 생태계를 재현하기 위해 정제된 규칙 기반 알고리즘 설계. 가상 전시회 3개 및 카테고리 5개(Beauty, Food, Living 등)를 설정하고, 각 참가사에 대해 고유한 **웹사이트 완성도(0~100)**, **이메일 신뢰지수(0~100)**, **수출 지원 여부(True/False)**, **직원수(가변)** 등 핵심 B2B 영업 징후 속성을 정형화하여 CSV로 출력.

### 2.2 가중치 기반 리드 가치 평가 모델 (Lead Scoring Layer)
* **파일명**: `scoring/lead_scoring.py`
* **수식 설계**: B2B 파트너십 소싱 가치가 높은 '우량 기업'을 선별하기 위한 100점 만점의 가중치 채점 알고리즘 구현.
* **가중치 매커니즘 (Weight Config)**:

$$\text{Lead Score} = \text{Web Quality} (30\%) + \text{Export Potential} (25\%) + \text{Scale Score} (20\%) + \text{Email Trust} (25\%)$$

  * **Web Quality (30점)**: 아웃바운드 영업 대상 기업의 디지털 성숙도를 평가.
  * **Export Potential (25점)**: 수출 경쟁력 보유 기업에 전용 가산점 부여.
  * **Scale Score (20점)**: 직원수 구간별 점수 차등 부여 (100명 이상: 20점 만점, 50명~99명: 15점, 10명~49명: 10점, 10명 미만: 5점).
  * **Email Trust (25점)**: 마케팅 수신 도달률 보장지수.

### 2.3 안전 준수형 순차 웹 크롤링 기법 (Crawler Layer)
* **파일명**: `data/real_crawler.py`
* **전술적 회피 및 보안 준수 방법론**:
  1. **SSL 보안 우회 가드**: `서울푸드(Seoul Food)`의 호스트 네임 불일치 예외를 탐지하여 무리한 연결 강행을 중단하고 즉각 안전 점프.
  2. **세션/액세스 권한 가드**: `K-Beauty Expo`의 참관객 비공개 영역 진입 제한(로그인 자바스크립트 얼럿)을 완벽히 인식하고 법적 가이드라인에 의거하여 스킵.
  3. **EUC-KR 디코딩 호환 및 DOM 정규 파싱**: 공공 접근이 허용된 `메가쇼(Mega Show)` 카탈로그에서 40개의 실존 제품 타이틀과 전시 브랜드를 BeautifulSoup으로 완벽 파싱 후 정제.

### 2.4 공간 정보화 매핑 및 UI 시각화 레이어 (Presentation Layer)
* **파일명**: `dashboard/app.py`
* **위치기반 매핑 방법론**: 대한민국 대표 산업 클러스터 12곳(성수 팝업 밸리, 판교 테크노, 해운대 물류, 제주 유기농 등)의 물리적 위경도 데이터를 설계. `company_id` 해시 함수를 통해 **결정론적이고 영구히 일관적인 소싱 지리적 위치** 부여.
* **미세 노이즈 분산(Jittering)**: 좌표가 정확히 같은 기업들이 겹쳐서 가려지는 현상을 막고자, ID 시드값 기반 미세 편차(Jitter) 수식 적용.

## 3. 핵심 산출물 및 인프라 현황

### 3.1 프로젝트 산출물 구조
프로젝트 루트 하위에서 일괄적으로 빌드 및 관리가 가능한 마스터 패키지입니다.

```text
kexpo-leadgen-lite/
├── data/
│   ├── generate_sample_data.py   # [산출물] 가상 리드 제너레이터 소스
│   ├── real_crawler.py           # [산출물] 실시간 순차 예외 처리 크롤러 소스
│   ├── exhibitions.csv           # [데이터] 가상 전시회 마스터 테이블
│   ├── companies.csv             # [데이터] 가상 기업 마스터 테이블
│   ├── crawled_exhibitors.csv    # [데이터] 메가쇼에서 직접 긁어온 실제 소싱 기업 테이블
│   └── lead_scores.csv           # [데이터] 가중치 기반 리드 등급 종합 계산서
├── scoring/
│   └── lead_scoring.py           # [산출물] 오프라인 리드 채점 연산 코어 소스
├── dashboard/
│   └── app.py                    # [산출물] 실시간 지도-데이터 융합형 Streamlit CRM 앱 소스
└── requirements.txt              # [인프라] 가동 패키지 디펜던시 마스터 리스트
```

### 3.2 온라인 실시간 배포망 정보
* **원격 소스 관리 저장소**: [GitHub Repository - project-2](https://github.com/bh-cmd/project-2.git)
